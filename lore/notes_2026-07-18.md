- **First GPU run was VOID** — owner flagged "wrong screen, definitely not 1920×1080." Root cause:
  the harness printed a HARDCODED `res=1920x1080` and only did `root.size=1920×1080`, which on this
  2-monitor `:0` (DP-2 2560×1440 + HDMI-1 1600×1200, both scale=1.0) leaves the OS window at Godot's
  **1152×648** default (window != viewport) — measured at the wrong resolution.
- **Hotfix v0.24.1 (planner, out-of-band from the factory — owner chose "patch now"):** place the
  window on a screen that fits the target (auto / `CARPET_PERF_SCREEN`), force
  `DisplayServer.window_set_size(1920×1080)` (+ `content_scale_factor=1`), **read the true size back**,
  fail-closed (`CARPET_PERF_RESULT FAIL`, exit 1) on any mismatch, print the real `res=WxH`. Confirmed
  the lever with a throwaway probe (method B: `window_set_size` → window==viewport==1920×1080; deleted
  the probe). Headless byte-identical (re-verified). VERSION→0.24.1, CHANGELOG entry.
- **VERIFIED result (true 1080p, gate enforced):** budget carpet 1.45M = **277 fps (4.6× the 60
  floor)**; full 2.4M hero = **180.6 fps (3.0×)**. Cost ~linear in points (budget only ~8% dearer per
  point than one dense block). **≤1.5M budget is generous; "perf constant unmeasured" risk RESOLVED; no
