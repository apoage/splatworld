## Decision surface (DECISIONS.md)

- **D2 (OPEN, now data-backed) — final foliage Gaussian budget.** The sweep proved the
  provisional gate **≤500k @ ≥20.7 dB is unachievable** for `pxl_144634`: 200k→16.88 ·
  350k→18.91 (knee) · 500k→19.51 · uncapped 2.39M→21.71 dB. 20.7 dB needs ~1.1–1.2M.
  **Recommendation (a):** relax the foliage PSNR floor, take ~350–500k + opacity-0.02 prune
  (fits ≤1.5M-whole-carpet across many instances). Owner picks. opacity-0.02 = the one
  useful prune lever (~−14% splats, ~0 dB); isolation/scale pruning is harmful, left off.
- **D3 (OPEN, gated to M4) — GDGS orientation convention.** `export` + GDGS's default
  −180° Z correction net to 180°-about-Y: up preserved, **azimuth yaw-flipped**, invisible
  under orbit but an identity-vs-scatter-basis inconsistency M4 must resolve Godot-side
  (always set an explicit node basis; do NOT re-derive the export matrix). Needs one A/B render.

## Where it stands / why I stopped before M2

M2 (`decompose`, GI-GS hybrid vendor+port) is **L/high** — it touches the schema contract
and the core relight thesis, front-loads a risky external CUDA build-verify (GI-GS on
sm_86/cu124), and is a multi-hour port. Launching it unattended at the tail of this run is
the wrong risk profile. It wants its own (ideally attended) session, and confirmation of the
vendoring scope. D1 already fixed the impl (GI-GS, MIT); the vendoring hard-rules are in the
task Notes (never copy the Inria rasterizer fork / nvdiffrast).
