## v0.19.0 — what the prototype gives you (the D7 evidence tool)

A `sign_mode` uniform in the relight pass, cycled live in the viewer with key **N** (HUD shows
the active mode). Formulas match `docs/d7-synthesis-2026-07-17.md` §4:

- **0 signed** — `max(dot(N,L),0)`, the shipped v0.18.0 path. **Byte-identical when off** (measured:
  render-gate means agree with restored v0.18.0 to the renderer's own ~1e-5 jitter).
- **1 sign-free WRAP** — `clamp((abs(dot(N,L))+w)/(1+w),0,1)/(1+w)`, w≈0.4. abs-FIRST, double-normalized
  (NOT half-Lambert, NOT plain abs). Live-tweak w via `,`/`.` while in mode 1.
- **2 flip-toward-camera** — `N' = dot(N,V)>=0 ? N : -N` then signed; V = splat→camera (camera world
  pos added to the binding-5 UBO, 208→224 B; push constant untouched).
- **3 confidence-blend — SKIPPED**: per-splat covariance scales are not in any buffer the pass binds,
  and extending the material buffer for a prototype is forbidden by the task Gate. (If mode 3 is
  wanted later, it needs a material-buffer extension = a schema/importer change = an owner call.)

Cost is ~free: signed 7.95 / wrap 7.76 / flip 7.75 ms @ 2.4M splats, 1080p.

New self-checks: `relight_sign_gate.gd` (closed-form per mode, each proven by fault injection — drop
the abs or the flip and the gate FAILs — plus a raw-invariance check) and `relight_sign_perf.gd`.

## v0.19.1 — slice 4, and what it unblocks

`render_orbit`/`render_sparkle` were rendering grounded `.relightply` heroes with the splat node left
at ≈identity, so GDGS's conditional −180°Z correction fired and flipped them upside down. Fixed by the
D3 rule (`gs.transform = Transform3D.IDENTITY` after `add_child`), mirroring the already-proven
`render_matrix.gd:210` pattern. Left untouched by design: `render_probe.gd` (vanilla cactus ply — needs
the correction), `relight_render_gate.gd` (aggregate-luma gate, orientation-invariant, green),
`render_foliage.gd` (vanilla plys).

Note on a red herring: the sign-gate's "IDENTITY-after-add_child does not rebuild the buffer" comment is
specific to its ±Z-invariant synthetic geometry (where it never actually needed the correction resolved).
`render_matrix`'s `sphere_consistency` gate empirically proves the pattern works on the real
scene-render path — **no D3 contradiction.**

**This code-unblocks slice-5 demo/gif regen** on grounded assets. I did NOT regenerate the footage:
right-side-up is a pixel judgment I can't make, and the demo video / README gif live in planner/owner
territory. That's the next step below.
