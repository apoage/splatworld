## What changed and why it's safe
- **The task's premise was wrong — corrected.** The task claimed per-splat world position was
  "already available to the pass." It is NOT: the pass binds only the screen-space culled buffer +
  the material/instance buffers. Fix: added per-Gaussian **object-space (centered) position** to OUR
  material GPU buffer (grown 2→3 vec4, 32→48 B; new `pos_label` slot, filled from the loader's
  already-parsed positions) and transform to world in the shader with the SAME
  `instance_model_matrices[si.x]` used for the normal. **NOT a PLY schema change** — `.relightply`
  already stores x/y/z; `ply_io.py`/`SCHEMA_VERSION` untouched. Runtime GPU-buffer layout only.
- **Shader:** additive point/spot term AFTER the RAW early-return (inverse-square × smooth range
  window × `smoothstep` cone), reusing the same albedo/normal/trans. Flashlight params in a **new
  binding-5 storage buffer** (`ivec4 meta` + fixed `FlashLight[4]`, N=1 now). **Push constant unchanged
  at 48 B / 3 vec4.** The `FlashLight[4]` array is the documented **N=2–4 fireball extension point**
  (adding lights = a CPU-setter change only, no shader/descriptor churn).
- **Viewer:** `F` camera-attached flashlight + `O` reference orb (engine-lit + a mirrored
  `SpotLight3D` so the orb stays honest in flashlight mode) + HUD `flash=`/`orb=`.
- **Invariants held:** no `godot/addons/gdgs/` edit (binding-5 is our own descriptor on our shader,
  same pattern as env binding 4); env-SH stays binding 4 (`|env-flat|=0.00014` byte-unchanged from
  v0.15.0); flashlight-OFF skips the whole flash block (contributes exactly 0); no-materials
  early-return byte-identical.

## The verification arc (fix cycle 1)
- **Original high-tier panel:** no BLOCKER. The highest-risk claim — centered-object-space × instance
  matrix = correct world position — was **refuted as a risk** (proven correct): smoke pos-slot
  max_err 0; the flow-verifier isolated the flash term and showed the lit blob's centroid **tracks**
  the light's world-x (no uniform brightening, no offset), including through a non-identity −180°Z
  instance matrix.
- **1 MAJOR found (fail-closed lens, by fault injection):** `relight_flashlight_gate.gd` was
  **fail-OPEN on the range/falloff term** — its only splat sat at range_win≈0.98, so deleting the
  range clamp shifted the result <2% and slipped under tolerance. This is the exact run-#7 fail-open
  class (a gate asserting a property it doesn't exercise). **Fixed:** added **P4** (range_win≈0.31,
  closed-form checked) + **P5** (out-of-range → ≈0, hard discriminator); `SETTLE`→40 (cold-cache
  stability); `TOL_ABS`→0.03. **Re-verified genuinely fail-closed:** re-injecting `falloff = inv_sq;`
  now FAILS via P4 (err 0.31 vs tol 0.03, 10× margin) AND P5; revert → PASS (shader byte-identical).
- **1 MINOR fixed:** cone `smoothstep(cos_outer, cos_inner, x)` was NaN when inner==outer (unreachable
  by current callers, live on the fireball extension path) → `cos_inner` clamped strictly above
  `cos_outer`.

## Operational notes for the next run
- **Stale-import gotcha (confirmed again, cost real time this run):** a `.glsl` edit needs a **clean**
  reimport — `rm -rf godot/.godot && ~/godot/godot --path godot --headless --import` — before any
  `--script` render. The mtime-based incremental `--import` does NOT recompile the changed shader and
  serves stale SPIR-V (both a verifier and the implementer hit false-fails from this).
- **One godot instance at a time on `:0`.** Concurrent renders cause GPU-contention false-fails
  (flash_delta reads 0). Honor the project's one-asset-per-GPU rule for the gates too.
- **API turbulence this run:** the implementer subagent died once on a transient 529 and repeatedly
  auto-resumed to run its own gates concurrently with the verifier (contention). It was stopped once
  independent verification was complete; the independent fail-closed judge's numbers are the
  authority (single-instance, closed-form-consistent).
