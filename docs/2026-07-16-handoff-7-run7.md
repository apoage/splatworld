## v0.15.0 relit-energy — what changed and why it's safe
- **Bug (owner, both heroes):** relit read as "bloom with extra saturation" — made-up light/shadow
  patches + hue shift. **Root cause:** `relight.glsl` applied the recovered env-SH ambient at weight 1.0,
  ignoring the ambient slider, while the sidecar coeffs are the FULL recovered capture illumination →
  relit ≈ unit sun + full capture light ≈ 4× energy (flow-verifier measured 4.01×; 23–27% of splats
  clipping).
- **Fix (runtime-side only):** `RelightPass.set_env_sh` scales all 9 c_lm by `1/(SH_C0·luma(c00))` so the
  sphere-mean luma of `ambient_sh(N)` == 1.0; `relight.glsl` multiplies the env branch by the ambient
  slider. The slider now drives env strength exactly like the flat fallback; the env contributes only
  directional shape + relative tint. `sh_env.py`, the `*_env_sh.json` sidecar bytes, and export's A_l·π
  folding are UNTOUCHED — exports stay engine-agnostic ground truth.
- **Gotcha for CI/next runs:** a `.glsl` edit needs `godot --headless --import` before a `--script`
  render, or the run uses the STALE cached import and silently masks the change.

## Next Ready — flashlight-orb (#3, now UNGATED) + architectural note
relit-energy shipping unblocked flashlight-orb (same shader seam, energy now calibrated). Deliberately NOT
started this run — it's a shading-contract change that deserves a fresh, careful start. **Heads-up the task
file states optimistically:** it claims "splat world position is already available to the pass." It is NOT
— the relight pass reads the CULLED (screen-space) buffer + the material buffer, neither of which carries
per-splat WORLD position. A point-light `L = normalize(flash_pos − splat_pos)` needs world position, so the
implementer must bind a GDGS means/position buffer (investigate its layout in `addons/gdgs/`) and transform
by `instance_model_matrices[si.x]`. Also: the push constant is FULL (48 B / 3 vec4) — the new light params
must go in a UBO (the task pre-authorizes this). Suggest scoping these two as the first implementation
decisions before writing shader code.
